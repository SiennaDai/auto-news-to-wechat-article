微信公众号文章 HTML 格式校验与修复。仅检查标签和样式规范，不修改文字内容。

## 检查项清单

### 一、HTML 结构规范

| 检查项 | 修复方式 |
|-------|---------|
| 存在外层 `<html>`/`<body>` 标签 | 移除 |
| 缺少根 `<div>` 包裹 | 用 `<div>` 包裹所有内容 |
| 输出中带 Markdown 标题（`# 标题`） | 转为 `<h1>`-`<h3>` |
| 输出中带 Markdown 加粗/斜体（`**`、`*`） | 转为 `<strong>`、`<em>` |
| 存在 `<b>` 标签 | 改为 `<strong>` |
| 存在 `<i>` 标签 | 改为 `<em>` |

### 二、标题层级规范

| 检查项 | 修复方式 |
|-------|---------|
| 存在 `<h4>`-`<h6>` | 降级为 `<h3>` |
| 标题层级跳级（h1 后直接 h3） | 将 h3 改为 h2 |
| 全文多个 `<h1>` | 保留第一个，其余改为 `<h2>` |

### 三、段落规范

| 检查项 | 修复方式 |
|-------|---------|
| 裸文本（不在 `<p>` 内） | 用 `<p>` 包裹 |
| `<div>` 内文字未用 `<p>` | 为裸文本加 `<p>` |
| 段落间多余空行 | 移除 |

### 四、`<strong>` 嵌套

| 检查项 | 修复方式 |
|-------|---------|
| `<strong>` 嵌套 | 去嵌套，保留一层 |

### 五、图片规范

| 检查项 | 修复方式 |
|-------|---------|
| 图片缺少 `image-wrapper` | 根据图片数量包裹对应布局容器 |
| `image-wrapper` class 不正确 | 修正为 single/side-by-side/carousel |
| 布局 class 与图片数量不匹配 | 按实际数量修正 class |
| `src` 路径格式不对 | 修正为 `images/文件名.jpg` |
| 缺少 `alt` 属性 | 从文件名提取 |
| 扩展名非 `.jpg` | 统一为小写 `.jpg` |
| 容器缺少 `aspect-ratio` | 补全 `aspect-ratio: 16 / 9` |
| 图片缺少 `object-fit: cover` | 补全 |
| side-by-side 容器缺少 `min-width: 0` | 补全 |

### 六、样式规范

| 检查项 | 修复方式 |
|-------|---------|
| `<p>` 缺少 `text-indent: 2em` | 补全，保留已有样式 |
| 容器缺少必要的内联样式 | 按布局类型补全样式 |

## 重要

- `data-el-type`、`data-el-id` 等 data-* 属性必须保留，不可删除
- 不修改文字内容，只修标签和样式
- 如有问题自动修复，没有问题原样输出

首行只输出 ok
第二行起输出完整HTML
