# Skill: News-to-Narrative Transformation

    ## 概述

    本skill用于将新闻报道转化为具有叙事性、可读性和传播力的纪实文章。转化过程强调信息重组、情感升华和结构优化，适用于活动总结、人物专访、项目回顾、成果汇报等多种场景。

    ---

    ## 一、输出格式规范

    ### 1. 基本要求

    - 输出内容为完整的 HTML 文档片段（无需包含 `<html>`、`<body>` 等外层标签）
    - 所有样式使用内联方式或标准 HTML 标签
    - 标题层级使用 `<h1>`、`<h2>`、`<h3>` 标签
    - 每个自然段落用 `<p>` 标签包裹
    - **每个 `<p>` 标签必须添加首行缩进样式：`style="text-indent: 2em;"`**
    - 段落之间用 `</p><p>` 分隔，不留空行
    - **输出内容必须以 `<div>` 开头，以 `</div>` 结尾**，整个内容被一个根 `<div>` 包裹

    ### 2. 输出示例

    <div>
    <h1>全校运动会圆满落幕</h1>
    <div class="image-placeholder" data-images="封面.jpg" data-layout="single"></div>
    <p style="text-indent: 2em;">10月15日，秋风送爽，全校师生齐聚体育场。</p>
    <h2>开幕式：青春飞扬</h2>
    <p style="text-indent: 2em;">国旗方阵率先入场，全场肃立。</p>
    </div>

    ## 一、Narrative的核心特征

    ### 1. 表述要求

    - **信息准确**：所有事实、数据、人名、时间、地点与原文一致，不可杜撰
    - **逻辑清晰**：事件按时间线或主题线有序组织，因果分明
    - **情感适度**：在客观事实基础上融入适度的情感渲染，但不夸张、不煽情
    - **主题突出**：每一段落有明确的中心思想，服务于整体主旨
    - **受众导向**：语言通俗但不失格调，符合目标读者的阅读期待

    ### 2. 行文风格

    - **整体基调**：庄重而不失温度，专业而不失亲和，正式而不失生动
    - **语言特征**：书面化、规范化表达；句长适中，节奏有变化；用词精准，避免空洞形容词堆叠；可适度运用排比、对偶等修辞
    - **人称视角**：以第三人称为主，辅以第一人称引语增强真实感
    - **时间感**：兼顾过去（事件回顾）、现在（现场描述）、未来（意义展望）

    ## 二、从News到Narrative的具体转化步骤

    ### 步骤一：信息提取

    从news中提取以下五类信息并分类记录：

    1. **核心事实**：时间、地点、主体、事件名称
    2. **数据信息**：人数、场次、时长、数量等可量化内容
    3. **人物信息**：发言人的直接或间接引语、关键人物的行为描述
    4. **背景信息**：历史渊源、前期筹备、组织架构、关联事件
    5. **意义阐释**：活动目的、成效总结、价值判断、未来展望

    ### 步骤二：主题提炼

    从提取的信息中确定一个核心主题：

    - 寻找news中出现的高频词或核心概念
    - 识别news结尾段或负责人发言中的核心表述
    - 将具体事件上升为一般意义

    核心主题应具备概括性、价值性和独特性。

    ### 步骤三：结构搭建

    根据news的内容特征选择以下结构之一：

    **结构A（单一事件或连续性活动）**：
    - 开篇：背景引入或主题定调
    - 主体：按时间顺序展开——筹备阶段、执行过程、后续工作
    - 结尾：成效总结、意义升华、未来展望

    **结构B（多场景并列活动）**：
    - 开篇：总体概述
    - 主体：每个场景独立成节，内部结构保持一致
    - 结尾：汇总共同意义

    **结构C（逻辑递进式）**：
    - 开篇：提出问题或设定主题
    - 主体：按逻辑层次展开——是什么、怎么做、有何成效
    - 结尾：回应开篇，点明深层价值

    ### 步骤四：段落组织

    每个段落内部按以下顺序组织信息：

    1. **主题句**：本段核心观点
    2. **事实支撑**：数据、时间、地点、人物行为
    3. **细节呈现**：使事实具象化的具体信息
    4. **过渡语句**：引出下一段或回扣本段主题

    ### 步骤五：开篇写作

    根据素材特点选择开篇方式：

    - **时间节点式**：以特定时间或季节切入，营造仪式感
    - **主题定调式**：首句直接点明事件的核心价值
    - **场景描绘式**：从一个具体画面或瞬间切入，渐次展开

    开篇应在一至两段内完成背景交代和主题定调。

    ### 步骤六：主体展开与衔接

    在主体部分，运用以下衔接技巧：

    - **承接**：使用因果关联词、逻辑递进词、场景转换词
    - **转折**：从事实转向意义、从局部转向整体、从现象转向本质
    - **并列**：多场景或多主体之间使用统一的转换标记

    各主体段落之间应有明确的逻辑关联，避免跳跃。

    ### 步骤七：细节植入

    在事实陈述中适时植入以下细节：

    - 关键人物的具体言行
    - 典型场景的画面描绘
    - 能够反映氛围的环境描写

    细节植入的原则：不改变事实、服务于主题、控制密度。

    ### 步骤八：结尾写作

    根据全文主旨选择结尾方式：

    - **成效总结式**：归纳直接成果和间接影响
    - **意义升华式**：将具体事件提升到理念或价值层面
    - **展望期许式**：表达对未来的期待，呼应开篇

    结尾应与开篇形成有机呼应，避免突兀收束。

    ### 步骤九：语言转化

    在语言层面进行以下转换：

    - 将客观陈述保留事实，增加适度的背景烘托
    - 将数据融入上下文，避免孤立罗列
    - 增加句长变化，使用长短句交替
    - 将口语表达转化为书面化、规范化表达
    - 保留直接引语，用于增强人物立体感

    ### 步骤十：重点内容标记

    在本步骤中，识别并标记文本中的重点内容，使用 `<strong>` 标签包裹。

    #### 重点内容的识别范围

    **类型一：具体的人物及其身份**
    - 格式为"机构/职务 + 姓名"或"姓名 + 机构/职务"
    - 抽象人物不要识别，如"学长学姐们"、"学弟学妹们"
    - 识别模式包括但不限于：
    - `[学院名称]学院[职务][姓名]`
    - `[职务][姓名][老师/教授/博士]`
    - `[年级]级[专业][姓名][同学/学长/学姐]`
    - 示例：
    - `创新创业学院张伟院长` → `<strong>创新创业学院张伟院长</strong>`
    - `2022级计算机专业李明同学` → `<strong>2022级计算机专业李明同学</strong>`

    **类型二：重要引语或核心理念**
    - 发言人的核心观点或金句
    - 活动主题或口号
    - 具有价值导向的表述
    - 识别特征：引号内的内容、排比句式、总结性判断
    - 示例：
    - `"始于感恩，成于传承，超越梦想"` → `<strong>"始于感恩，成于传承，超越梦想"</strong>`
    - `这标志着我校在这一领域实现了零的突破` → `<strong>这标志着我校在这一领域实现了零的突破</strong>`

    **类型三：关键数据或成果**
    - 具有标志性意义的数据
    - 首次实现、突破性进展
    - 获奖情况或荣誉

    #### 标记密度控制

    - 每个自然段落中的 `<strong>` 标签不超过5处，不少于2处

    #### 标记示例

    ```html
    <p>活动邀请了<strong>创新创业学院张伟院长</strong>出席并致辞。</p>
    <p>张院长在发言中强调："<strong>创新不是天马行空，而是脚踏实地解决问题</strong>"。</p>
    <p>最终，<strong>全校共有127支队伍报名参赛，创历史新高</strong>。</p>
    ```

    ### 步骤十一：整体润色

    完成初稿后进行以下检查：

    - 全文基调是否统一
    - 段落衔接是否自然
    - 是否有语病或错别字
    - 情感渲染是否适度
    - 核心主题是否贯穿全文
    - HTML 标签是否正确闭合（检查 `<h1>`、`<h2>`、`<p>` 等）
    - 重点标记是否使用 `<strong>` 而非其他标签

    ## 三、转化过程中的注意事项

    ### 需要避免的问题

    - 逐句改写而非重组信息
    - 按时间顺序无重点地罗列所有细节
    - 大量使用空洞的形容词堆叠
    - 为追求文采而改变事实
    - 全文风格前后不一致
    - 使用 Markdown 语法（如 `#`、`**` 等）替代 HTML 标签
    - 使用 `<b>` 标签替代 `<strong>`（语义化要求）

    ### 质量自检要点

    - 所有关键事实是否准确涵盖
    - 是否有清晰的开头、主体、结尾结构
    - 语言是否符合书面化、专业化标准
    - 情感表达是否以事实为基础
    - 核心主题是否得到有效凸显
    - 标题是否使用 `<h1>`/`<h2>` 标签而非 `#` 标记
    - 重点人物、引语是否已用 `<strong>` 标记
    - 整个输出是否有一个根 `<div>` 包裹全部内容
